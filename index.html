  • 类的成员的值由下到上依次传递过去。

函数传参-类对象的引用

  • 使用了引用之后，直接传递对象地址。

类对象作为函数返回值

  • 当函数返回某个类的对象时，如果寄存器能够保存对象的所有成员的数据时。会将所有数据传进寄存器里。
  • 否则，调用方会预先开辟这个类对象的栈空间，将对象指针传进去。
  • 如果函数生成的对象是是局部变量，切忌返回对象的指针或引用。若要返回对象的指针或引用，需要在堆区进行分配，即使用new来生成。

C++反汇编分析-数组

发表于 2022-10-22

编译器：x86-64 gcc 12.2

一、栈空间分配

  • 数组由低地址往高地址分配内存，即小端模式。
  • 可以给超出下标的数组赋值，这会导致栈帧以外的数据被修改。
  • rsp以外的128字节被当作保留区（red zone）,可以用于临时值的存储。即如果使用空间小于128字节，就不需要移动rsp。

二、字节对齐

  • 113字节的大小的数组占用了129字节的栈空间。这是因为System V ABI的规范，需要16字节对齐。
  • rsp向下拓展了8字节。由于red zone只有128字节，所以rsp需要向下拓展，从而扩充整体的栈空间。根据System V ABI的规范，rsp的值要求是8n+16。
  • 总之，当数组大小n超过112字节的时候，rsp需要移动x=ceil(n/16)*16-128+8 字节。 例如n=1000,x=888。

三、栈空间最大值

  • 数组能够占用的栈空间最大是2^63-1字节。超过这个值会编译报错。这取决于rsp寄存器的位数。
  • 如果是32位，则不能超过2G(2^31-1)。

四、堆空间分配最大值

  • 同理，数组在堆能够分配的最大值是取决于rax寄存器的位数，因为调用new函数需要的传参的立即数需要存放在rax。只要n不超过rax寄存器的位数，是可以编译成功的。
  • 但在运行时往往是达不到理论上的最大值。动态开辟内存时，会使用 malloc 等函数在堆区开辟空间，分配器将通过特定的数据结构（如链表）组织堆区的数据块。如果内存不足，则会触发std::bad_alloc函数。

五、全局变量

  • 全局变量在编译期已经确定了地址，可以通过当前地址（rip）+偏移（立即数）进行访问。
  • 同理，全局变量数组空间的理论最大值也取决于寄存器的位数。
  • 全局静态变量与全局变量类似。差别在于无法被外部文件所访问。

六、局部静态变量

  • 局部静态变量预先会被当作全局变量处理，差别在于作用域和声明初始化赋值操作。
  • 如果是通过变量对局部静态变量进行声明且赋值，需要使用1个标志位来判断局部变量是否初始化过。
  • 局部静态变量若用常量赋值，则在编译期完成。

七、char数组初始化字符串

  • 字符串拷贝每次复制8个字节到寄存器（64位），再复制到内存。小于8字节，则是立即数复制到内存。
  • 字符串结尾会添加一个0。

docker-文件系统

发表于 2022-10-22

Docker 支持了不同的存储驱动，包括 aufs、devicemapper、overlay2、zfs 和 vfs 等等，在最新的 Docker 中，overlay2 取代了 aufs 成为了推荐的存储驱动，但是在没有 overlay2 驱动的机器上仍然会使用 aufs 作为 Docker 的默认驱动。

overlayfs

  • Overlayfs 是一种类似 aufs 的一种堆叠文件系统，于 2014 年正式合入 Linux-3.18 主线内核，

  • 它依赖并建立在其它的文件系统之上（例如 ext4fs 和 xfs 等等），并不直接参与磁盘空间结构的划分，仅仅将原来底层文件系统中不同的目录进行“合并”，然后向用户呈现。

docker应用

overlay分为lower,upper,merge。

  • lower（镜像层）是只读层，用于存放docker共享的image镜像。
  • upper（容器层）是读写层，通过commit操作层层堆叠覆盖。
  • merge 是两者的合并，做为容器的rootfs。

关于docker commit

  • docker commit，实际上就是对运行起来的docker容器，把upper的可读写层，加上原先lower的容器镜像的只读层，打包组成了一个新的镜像。
  • lower的只读层在宿主机上是共享的，不会占用额外的空间。
  • 在容器里对镜像 rootfs 所做的任何修改，都会被操作系统先复制到upper的可读写层，然后再修改，也就是Copy-on-Write。
  • 在容器里对lower层文件的删除操作是软删除。通过whiteout进行遮挡。因为lower是只读的。

System V ABI x86-64

发表于 2022-10-21

System V

System V应用程序二进制接口是一组规范，详细说明了调用约定，目标文件格式，可执行文件格式，动态链接语义等等。现在，它已成为主要的Unix操作系统（例如Linux，BSD系统和许多其他操作系统）使用的标准ABI。 可执行和可链接格式（ELF）是System V ABI的一部分。

寄存器相关

  • 函数的参数在寄存器rdi，rsi，rdx，rcx，r8，r9中传递，并且其他值以相反的顺序在堆栈中传递。前6个从左到右依次放入rdi，rsi，rdx，rcx，r8，r9，超出6个的参数从右向左放入栈中。

  • 使用call指令来调用函数，该指令将下一条指令的地址压入堆栈并跳转到操作数。 被调函数使用* * ret指令返回调用者，该指令从堆栈中弹出一个值并跳转到该值。

  • 在调用调用指令之前，堆栈是16字节对齐的。
    比如：

    x86-64 gcc 12.2 汇编结果

  • 函数保留寄存器rbx，rsp，rbp，r12，r13，r14和r15； rax，rdi，rsi，rdx，rcx，r8，r9，r10，r11是暂存寄存器。

  • 返回值存储在rax寄存器中，或者如果它是128位值，则高64位进入rdx。

栈帧（stack frame）

每个函数在运行时都维持一个栈帧。从高地址向低地址拓展。如下图所示。

  • 栈帧末端需要与16字节对齐。**%rsp + 8总是16的倍数。**这是因为传统rdp作为帧指针的技术是不可以使用rsp来索引栈帧的。通过这个规定可以使得这两个寄存器都能作为帧指针。

  • %rsp所指向的位置之外的128字节的区域被认为是保留的。被认为是保留区（red zone），不得被信号或中断处理程序修改。这是因为128字节内的位置可以用一个字节的偏移来寻址。因此，函数可以将这个区域用于临时数据，而这些数据在整个函数调用中是不需要的。

docker网络通信原理简单分析

发表于 2022-10-21

概述

docker容器可以直接使用宿主机的网络栈（–net=host），即：不开启 Network Namespace，比如：

在这种情况下，这个容器启动后，直接监听的就是宿主机的 80 端口。

而所谓“网络栈”，就包括了：网卡（Network Interface）、回环设备（Loopback Device）、路由表（Routing Table）和 iptables 规则。

像这样直接使用宿主机网络栈的方式，不可避免地引入共享网络资源的问题，比如端口冲突。

所以，在大多数情况下，我们都希望容器进程能使用自己网络栈，即拥有属于自己的 IP 地址和端口。

那么，相互隔离的容器进程是怎么样进行通信的呢？

网桥

我们可以把容器当作一台主机。

两台主机之间的通信，就是利用网线连接起来。

多台主机的通信，则是将网线连在同一台交换机上。

在Linux中，网桥（Bridge）可以起到虚拟交换机的作用。

Docker服务会在宿主机创建一个名为docker0的网桥，所有docker都可以通过它来进行通信。

Veth Pair

veth pair通常被用于连接不同网络命名空间的虚拟网线。
veth pair也可以用于连接容器和宿主机的docker0网桥。比如：

可以看出veth pair的一端是容器的eth0，另一端是宿主机的veth1a2cc8c。

总结

docker通过docker0和veth pair实现了链路层上的网络通信。
docker0可以充当二层交换机的作用，veth pair起到“网线”的作用。

C++文件内存

发表于 2022-10-21

可重定位目标文件内存

  • .text 代码段
  • .rodata 只读数据，如字符串常量。
  • .data 全局变量和静态变量。static和const修饰属于内部链接，不会放到导出符号表。
  • .bss 未初始化的全局变量和静态变量。只分配大小，不会分配内存。可以缩减可执行文件的大小。
  • .symtab 导出符号表： 本编译单元所有定义好的符号和地址，用于外部链接。
  • .rel.text 记录一些.text位置的列表，未定义的引用，未定义符号的地址。比如extern修饰的声明；
  • .rel.data 需要地址重定向的数据

头文件变量相关

  • const变量在编译时已经变成了常量，不会有符号名冲突（链接期）。
  • 非const变量link可能会冲突。引用了其头文件时，等同于在源文件中声明或定义了这个全局变量。
  • 用extern声明全局变量 ，即使没有引用该头文件，也可以使用。
  • static变量不会冲突（连接性为内部）。
  • 命名空间中声明的名称的连接性为外部的。
